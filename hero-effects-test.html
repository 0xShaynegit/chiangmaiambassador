1. CSS Floaters

What it does: Pure CSS @keyframes animation. Simple glowing orbs float upward with opacity fades.

✓ Pros: Smallest file size. Zero JS overhead. GPU accelerated. Perfect for subtle background.

✗ Cons: Limited visual complexity. Fixed path (always straight up). Can't respond to user input.

File size: ~500 bytes CSS

Performance: Excellent (60fps on mobile)

2. Canvas Particles

What it does: HTML5 Canvas with vanilla JS. Renders particles with physics simulation. Responds to mouse movement (optional).

✓ Pros: Beautiful particle effects. Can add physics (gravity, wind, collision). More dynamic and interactive.

✗ Cons: Requires JS animation loop. Uses more CPU. ~3-4KB JS code needed.

File size: ~3KB JS + canvas context

Performance: Good (60fps on desktop, 30-40fps on mobile)

3. SVG Lanterns

What it does: Inline SVG lantern shapes with vanilla JS animation. Creates realistic lantern silhouettes with glow.

✓ Pros: Vector quality (scales perfectly). Can animate SVG properties (rotate, opacity). Thematic and beautiful.

✗ Cons: Larger SVG markup. More DOM elements. Requires JS for position updates.

File size: ~4KB SVG markup + 2KB JS

Performance: Good (60fps on desktop, good on mobile)

My Recommendation for Chiang Mai Ambassador

Use CSS Floaters (#1)

Why: Keeps the site ultra-lightweight, truly portable, zero dependencies. The subtle glow effect is elegant without being distracting. Maintains our vanilla-first philosophy perfectly.

OR combine #1 + #2: CSS for most floaters, Canvas for a subtle particle system on desktop only (graceful degradation on mobile).
